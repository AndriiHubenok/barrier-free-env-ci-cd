# Dockerized-Barrier-free-environment-complete

**Dockerized-Barrier-free-environment-complete** — це проєкт, що дозволяє запустити повноцінне середовище для роботи з **PLM_Router**.  
Система складається з декількох взаємопов’язаних сервісів, кожен з яких відповідає за окрему частину функціональності.

## Компоненти проєкту

### 1. Barrier-free-environment-diploma (Location-service)
Бекенд-сервіс, що містить основну бізнес-логіку роботи з:
- локаціями,
- користувачами,
- критеріями оцінювання локацій.

📌 **Для запуску** необхідно додати відповідний `.env` файл у корінь папки сервісу.

---

### 2. Barrier-free-environment-web
Фронтенд веб-застосунку, розроблений за допомогою **Angular**.  
Відповідає за:
- відображення карти,
- побудову маршрутів,
- відображення локацій.

✅ Працює одразу після запуску та **не потребує додаткових налаштувань**.

---

### 3. image-storage-service
Сервіс, що відповідає за:
- збереження зображень,
- роботу з ними (отримання, обробка тощо).

📌 **Для запуску** необхідно додати відповідний `.env` файл у корінь папки сервісу.

---

### 4. py_nsfw-filter_online
AI-сервіс, який аналізує зображення на наявність:
- нецензурних,
- небажаних елементів.

Використовується для фільтрації контенту, що завантажується користувачами.

---

### 5. qgis_server
Сервіс, який запускає середовище **QGIS** та:
- очікує звернень до моделі,
- будує маршрут до найближчої локації з **найменшою кількістю бар’єрів**.

📌 **Для запуску** необхідно додати відповідний `.env` файл у корінь папки сервісу.

---

### 6. spring-ai-text-moderation
Текстова AI-модель, що контролює всі текстові звернення користувачів у застосунку, зокрема:
- додавання локацій,
- оновлення локацій,
- створення коментарів.

Модель перевіряє:
- цензурність повідомлень,
- відповідність тематиці.

На основі перевірки ухвалюється рішення — **опублікувати повідомлення** або **відхилити його**.

📌 **Для запуску** необхідно додати відповідний `.env` файл у корінь папки сервісу.

---
